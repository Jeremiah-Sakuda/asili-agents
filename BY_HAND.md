## 1) Create MongoDB Atlas M0 cluster, least-privilege DB user, network access, get `MONGODB_URI`

All in the Atlas web console (https://cloud.mongodb.com) — none of this can be scripted on the free tier.

1. **Project/cluster:** Create (or pick) a project → **Build a Database** → **M0 (Free)**. Provider/region: AWS, choose a region near `us-central1` (e.g. `us-east-1` N. Virginia) to minimize latency to Cloud Run. Name the cluster `asili` (or accept default). Wait until status is green (~3–5 min).
2. **Database user (least-privilege):** Security → **Database Access** → **Add New Database User** → Authentication = Password.
   - Username: `asili_app`
   - Password: click **Autogenerate Secure Password** → **copy it now** (you cannot see it again).
   - **Database User Privileges → Specific Privileges**: grant **`readWrite`** on database `asili` only (do **not** grant `atlasAdmin` or cluster-wide roles). The running app reads through MCP `--readOnly`, but the one-time seed needs write; scoping to the single DB keeps it least-privilege.
3. **Network access:** Security → **Network Access** → **Add IP Address**.
   - For the seed from your laptop: **Add Current IP Address**.
   - For Cloud Run (no static egress IP on the free path): add `0.0.0.0/0` **(Allow access from anywhere)**. M0 has no PrivateLink/peering, so this is required for Cloud Run to connect. Acceptable for a demo because the user has no admin rights and the app connects read-only; **delete the `0.0.0.0/0` entry after judging.**
4. **Get the SRV string:** Cluster → **Connect** → **Drivers** → copy the `mongodb+srv://...` URI. Replace `<password>` with the autogenerated password, and append the db name + read preference. Final value to use as `MONGODB_URI`:
   ```
   mongodb+srv://<DB_USERNAME>:<URL_ENCODED_PASSWORD>@<CLUSTER_HOST>.mongodb.net/asili?retryWrites=true&w=majority&appName=asili
   ```
   - **URL-encode** any special characters in the password (`@`→`%40`, `:`→`%3A`, `/`→`%2F`, `#`→`%23`). If autogen produced symbols, regenerate a long alphanumeric-only password to avoid encoding bugs.
   - Keep `MONGODB_URI` only in: your local `.env` (gitignored), GitHub Actions secrets, and Cloud Run env. **Never commit it.** Confirm `.env` is in `.gitignore` before pasting.

```bash
URL=$(gcloud run services describe asili-agents --region us-central1 --format 'value(status.url)')

# 1. Service is up and NOT in demo mode

# 4. Create a conversation, run the team
CID=$(curl -s -X POST "$URL/api/conversations" | python -c "import sys,json;print(json.load(sys.stdin)['id'])")
curl -s -X POST "$URL/api/run" -H 'content-type: application/json' \
  -d "{\"conversation_id\":\"$CID\",\"message\":\"Do you have the purple tea in stock? Can you do a bundle?\"}"
#    -> draft mentions 6 tins, ~$34 bundle, margin held; steps have grounded_facts/sources
# 5. Baseline lies (for contrast)
curl -s -X POST "$URL/api/run/baseline" -H 'content-type: application/json' \
  -d "{\"conversation_id\":\"$CID\"}"        # has_tools:false, grounded:false
# 6. Trust Scorecard runs
curl -s -X POST "$URL/api/eval"      # team beats baseline on hallucination/margin/grounded
# 7. Approval gate works
curl -s "$URL/api/pending/$CID"      # has_pending true with a draft
curl -s -X POST "$URL/api/approve" -H 'content-type: application/json' \
  -d "{\"conversation_id\":\"$CID\",\"action\":\"approve\"}"   # status ok
# 8. Reset so the recorded demo is clean
curl -s -X POST "$URL/api/reset"     # {"status":"reset"}
```
